# Contributing to myClick

Thank you for looking. The most valuable thing you can do for myClick is **audit it** — read the code, check the cryptographic protocol, and verify that the privacy claims hold.

## What we welcome

- **Security disclosures** — via **security@myclick.camera** (see [SECURITY.md](SECURITY.md)). The highest-value contribution.
- **Protocol-specification feedback** — if the cryptographic design has a flaw, we want to know before it ships.
- **Privacy-claim verification** — if the code doesn't match what we say in public, tell us.

## What we're not taking (during v1.0)

- **Code contributions.** The engineering is closed during the founding period. This isn't community hostility — it's that the value of open-sourcing myClick is *auditability*, not crowd-sourced development. A privacy-critical codebase handling the biometrics of children is not a place for casual pull requests.

We may open up to code contributions after v1.0 stabilises. Until then: read, verify, and disclose.

## Voice

If you contribute documentation or disclosures, match the project's voice — plain English, honest about limits, no marketing-speak.
